# Project Retrospective – Vintage & Thrifted Clothing Store Website

## Overview
This Retrospective reflects the development process of the Vintage Clothing Store website. The purpose for this document is to evaluate and review its performance, identify weaknesses and strengths to document improvements on future projects. 

## What Went Well

### 1. Website Planning
The inital planning required a scope statement, work breakdown structure, and risk management plan. This helped in guiding the project in the right direction and keep the focus. Improvements included time management that prevented any further delays.

### 2. Version Documentation

Using GitHub provided the benefit to track progress and track documentation changes. Minimzing the risk of data loss and debugging and evaluation easier.

### 3. Layout and design

The final layout and design is visually appealing and user-friendly. Clear navigation, product categorization and responsivness improves usability across different devices.

### 4. Development and Testing
all features were tested in stages early for improvements, reducing errors upon completion

---

## What Went Wrong

### Change in Direction for this Project

A challenge that was encountered during this project was an overall shift in the topic. The inital plan was to create a website based on music genres that were my personal favorite. However, a different approach was taken to improve the quality, relevance and technicality of this project. This brought delays in which a complete redesign of the project scope, layout, and documentation had to be performed. This required more time to resturcutre content, and development planning. Despite the delay, the project stayed within required guidlines and objectives. This new and improved topic ultimatley lays a stronger foundation for demostrating skills on development and project management.

---
## What Could Be Done Differently

Allocating for more time on the shift in topics. The intial topic had uncertainity on if it would perform well which was the cause for the shift. Gathering feedback from other individuals could have helped identify a path more easily at first on decsions for a topic and improve time quaility.

---

## Lessons Learned

This project demonstrated that planning must be thorough, manage time qualitatively, and identify the risks. While setbacks occured and introduced a major challenge, they were valuable on learning to improve performance for future projects and oppurtunities.
